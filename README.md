# Pixel Pulse Analytics

Pixel Pulse Analytics is the independent systems studio of Amy Laird.

Positioning: **messy data, broken workflows, stale websites, and scattered information turned into clear systems people can actually use.**

The site now presents Pixel Pulse Analytics as a high-capability solo systems studio with practical fixed-scope entry offers. The goal is to avoid sounding like a low-end gig shop while still giving buyers concrete, low-friction ways to start.

## Current site strategy

The site has two levels:

1. **Start Small** — fixed-scope services that solve one real problem quickly.
2. **Bigger Systems Work** — audits, implementation sprints, and ongoing systems partnership for deeper operational cleanup.

This preserves the ceiling of serious consulting while making the first purchase easier.

## Core message

## Tech stack

Plain static HTML and CSS.

No framework. No npm. No build step. No dependencies beyond hosted Google Fonts.

This is intentional. The site is small, fast, easy to maintain, and deploys through GitHub Pages without creating a maintenance goblin that must be fed JavaScript updates forever.

## Design direction

The original editorial identity remains, but the redesign is more direct and service-forward.

- **Typography:** Fraunces for high-signal headings, Inter for readable body copy, JetBrains Mono for labels and prices.
- **Palette:** warm paper, deep ink, muted teal, blue, gold, and rust accents.
- **Layout:** strong hero, fixed-scope service cards, systems-studio positioning, proof of range, and clear intake process.
- **Tone:** sharp, practical, high-trust, and grounded. Not Fiverr. Not agency fog. Not AI hype sludge.
- **Mobile:** simplified navigation, full-width CTAs, stacked cards, and readable service metadata on small screens.

## Deployment

The repo is configured for GitHub Pages with the custom domain:

```txt
pixelpulseanalytics.com
```

`robots.txt` points crawlers to the sitemap, and the sitemap includes the homepage, contact page, and all six service pages.

## Next improvements

Recommended next iterations:

1. Add an Open Graph preview image.
2. Add a favicon.
3. Add `LocalBusiness` or `ProfessionalService` schema markup.
4. Add a lightweight contact form only when email intake becomes annoying.
5. Add anonymized before/after examples once real client work exists.
6. Add testimonials only when they are real. Fake testimonials are trust poison with better formatting.

## Brand boundary

Do not let the site drift into low-end task work. The services are small, but the brand is not.

The correct framing is:

> Serious systems thinking, packaged into practical starting points.
